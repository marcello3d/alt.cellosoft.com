Introduction

The Alt Cloud is designed to let Alt scale and handle increasing demand by increasing the number of servers running Alt.

Peer-to-Peer

Each Alt server node runs the exact same Alt platform application without the need for configuration.

The node generates a private and public key for identifying and encrypting communication between Alt nodes. Using the Alt control panel, you inform any Alt node of another Alt node's public key and they will become partnered.

Once two nodes are partnered they can communicate with any other Alt servers they are aware of.

Network connectivity, execution speed/memory is discovered and mapped between the network. When direct/secure pipes exist, the servers communicate with standard unencrypted streams, otherwise a ssh tunnel is used.

Database sharding

From there, database and execution is partitioned and duplicated between servers.

Since the app code and configuration is stored in the database, this gets duplicated across the network as well.

Yay big happy family.
